Server Management – Windows & Hybrid

NET-2220 pulled a single Windows Server VM all the way from initial install to “ready for production”: deployment, Active Directory, security hardening, patching, backup, and daily monitoring.

Windows Server 2019/2022 Active Directory CIS Hardening Backup & Automation

Course Outcomes

By the end of this class I could:

  • Deploy a Windows Server VM, set static networking, and promote it to an AD DS domain controller.
  • Design and implement OUs, users, and groups that support least-privilege access.
  • Apply CIS-style security baselines for password policy, lockout, firewall, services, and patching.
  • Configure Windows Server Backup and test restore operations to prove recoverability.
  • Automate server health checks with PowerShell and Task Scheduler for daily operational monitoring.

Milestones

Milestone 1

Deployment & Directory Configuration

Built SERVER3 as a domain controller for corp.local: static IP, hostname, AD DS role, forest creation, and OU structure (Users / Groups). Verified logon using a test account (jsmith) and membership in an administrative group.

Open Milestone 1 PDF Deployment & AD notes
Milestone 2

Security Hardening & Patch Management

Applied a Level-1 style CIS baseline: strong password / lockout policies, firewall on all profiles with only required ports open, and unnecessary services (Remote Registry, Telnet, Print Spooler) disabled. Verified patch state and audit policies through PowerShell and Event Viewer.

Open Milestone 2 PDF Hardening & patching notes
Milestone 3

Backup & Automation Implementation

Enabled Windows Server Backup to an external disk, scheduled regular backups, and ran a restore test into C:\TestRestore. Built ServerMonitor.ps1 to generate a daily health report (services, event logs, top CPU processes) and scheduled it via Task Scheduler.

How this maps to real work

Taken together, these milestones mirror what an entry-level systems administrator or junior infrastructure engineer does in a real environment: build a domain controller, lock it down to meet security baselines, patch and monitor it, and prove that backups and restores actually work.

4. Security Configuration Summary

4.1 Password & Account Lockout Policies

Account policies were configured to align with CIS Windows Server 2019 Level 1 recommendations.

PolicyValue
Minimum password length12 characters
Maximum password age60 days
Password history24 remembered passwords
Password complexityEnabled
Account lockout threshold5 failed attempts
Lockout duration / reset15 minutes

Configured via Local Security Policy → Account Policies → Password Policy and Account Lockout Policy.

4.2 Firewall Configuration

All firewall profiles were enabled, and inbound access was restricted to only what is required:

ProfileStatus
DomainEnabled
PrivateEnabled
PublicEnabled
  • RDP (TCP 3389) allowed only from local subnet or lab network.
  • Unnecessary inbound rules reviewed and disabled.
  • “Block all inbound except allowed rules” enforced by profile.

Profiles and rules were confirmed in Windows Defender Firewall with Advanced Security and via PowerShell cmdlets such as Get-NetFirewallProfile.

4.3 Disabled Services

To reduce attack surface, several legacy or unnecessary services were disabled:

ServiceReason
RemoteRegistry Prevents remote modification of the Windows registry.
Telnet (TlntSvr) Legacy plaintext protocol; replaced by SSH/modern tooling.
Print Spooler Disabled since no printing role is required on the domain controller.

Configured using Set-Service -StartupType Disabled for each service and verified in services.msc and via PowerShell.

4.4 Patch Level

Patch status was verified as of December 2025 using:

Get-HotFix | Sort-Object InstalledOn -Descending | Select-Object -First 10
systeminfo | Select-String "KB"

Windows Update history confirmed that the latest cumulative security updates had been applied. The server is fully patched as of the end of the course.

5. Backup and Recovery

5.1 Backup Configuration

Windows Server Backup was used to implement a full server backup regime.

Backup TypeFrequencyDestination
Full Server BackupDaily at 02:00 AMBackupDisk (E:)

Backup was scheduled using wbadmin and confirmed with:

wbadmin enable backup -addtarget:E: -include:C: -allCritical -schedule:02:00 -quiet

Initial backups were run manually using:

wbadmin start backup -backuptarget:E: -include:C: -allCritical -quiet

Backup status and events were validated in Event Viewer under Applications and Services Logs → Microsoft-Windows-Backup.

5.2 Recovery Validation

Available backup versions were listed with:

wbadmin get versions

Entries confirmed that full server and system-state backups are present and can be used for recovery. Backup completion and catalog integrity were verified in Event Viewer.

Even without restoring the entire server, this capstone confirms a documented, repeatable backup configuration, with clear steps for full restore if needed.

6. Automation and Monitoring

To support ongoing health checks, I created a PowerShell monitoring script and scheduled it via Task Scheduler to generate daily status reports.

6.1 Monitoring Script – ServerMonitor.ps1

Location: C:\Scripts\Monitoring\ServerMonitor.ps1

Outputs: C:\Scripts\Monitoring\Logs\Status_<timestamp>.txt

$timestamp = Get-Date -Format "yyyy-MM-dd_HH-mm"
$report = "C:\Scripts\Monitoring\Logs\Status_$timestamp.txt"

Add-Content $report "=== System Health Check ($timestamp) ==="
Get-Service | Where-Object {$_.Status -ne 'Running'} | Out-File -Append $report
Get-EventLog -LogName System -Newest 20 | Out-File -Append $report
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10 | Out-File -Append $report

The report includes:

  • All services that are not currently running.
  • The 20 most recent System log entries.
  • The top 10 CPU-consuming processes.

6.2 Scheduled Task

A Task Scheduler entry named Server Monitoring Daily was created:

SettingValue
TriggerDaily at 08:00 AM
Actionpowershell.exe -File C:\Scripts\Monitoring\ServerMonitor.ps1
Run AsAdministrative account

Running the task manually and reviewing the generated log file confirmed that automation is functioning and producing usable status reports for operations or SOC review.

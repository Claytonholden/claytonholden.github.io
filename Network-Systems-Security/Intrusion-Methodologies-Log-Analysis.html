NET-2650 • Conceptual Project

Intrusion Methodologies & Log Analysis

This project focuses on understanding common intrusion methodologies and how log data can be used to identify, document, and analyze suspicious activity in a controlled lab setting. The work emphasized evidence handling, timestamps, system identification, and repeatable documentation practices across both Windows and Linux environments.

Log Analysis Windows Event Logs journalctl Incident Response Evidence Handling

Overview

Through this project, I practiced documenting system evidence in a way that supports provenance, clarity, and repeatability. This included capturing hostnames, operating system details, and timestamps from both Windows and Linux virtual machines to establish when and where evidence was collected.

What I Learned

  • How intrusion activity can be investigated through host-based logs and system artifacts
  • Why timestamps, hostnames, and OS details matter in evidence documentation
  • How Windows and Linux logging approaches differ in structure and workflow
  • How to document findings clearly for repeatable analysis and reporting

Tools / Skills

  • Windows system information and event log awareness
  • Linux terminal commands and log inspection basics
  • Evidence collection and documentation
  • Professional reporting in a lab environment

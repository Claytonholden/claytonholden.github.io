2. Executive Summary

This report summarizes my work in CYBR-2102 across three main areas: designing a defense-in-depth network, detecting hostile activity in packet captures, and responding to an RDP-based incident using NIST SP 800-61. I built a segmented Packet Tracer environment with external, DMZ and internal zones; enforced least-privilege access using Cisco IOS ACLs and host firewalls; analyzed traffic in Wireshark for ping sweeps, TCP port scans and ICMP floods; and documented a full incident-response workflow including containment, eradication, and recovery.

All controls are mapped to CIS Controls v8 and NIST SP 800-61 where appropriate, and the attached evidence shows configuration screens, packet captures, and automation used to monitor and harden the environment.

3. System Architecture & Defense Layers

The Packet Tracer topology follows a classic three-tier design: external network → perimeter firewall/router → DMZ → internal LAN. Each zone has explicit ACLs and host-level controls instead of relying on a single perimeter rule.

3.1 Topology and Zones

3.2 Perimeter & DMZ Controls

The edge router/firewall enforces least privilege between zones. Only HTTP from the Internet to the DMZ web server is allowed; direct Internet → internal access is denied. Administrative access to the router is restricted to a management subnet.

3.3 Host & Management Plane Protection

DMZ and internal hosts use local firewalls to restrict inbound management ports and to log unauthorized access attempts. Router and switch management is secured with SSH and role-based access.

4. Monitoring & Threat Detection

Wireshark was used to capture and analyze several simulated attacks: an ICMP ping sweep, a TCP SYN scan, and a high-rate ICMP flood. Display filters allowed me to isolate each pattern and classify it as malicious or suspicious.

4.1 Tools & Methods

Primary tool: Wireshark with filters like icmp and tcp.flags.syn == 1 and tcp.flags.ack == 0. Simulated attacks were generated from a dedicated “attacker” host in Packet Tracer.

4.2 Secure Remote Access Monitoring

SSH was configured as the secure management protocol. Captures confirmed encrypted sessions and logged both failed and successful logins.

5. Incident Response Summary

Milestone 3 applied NIST SP 800-61 to a simulated “Unauthorized RDP brute-force and data exfiltration attempt” on an internal Windows server. Logs showed repeated Event ID 4625 failures followed by suspicious outbound traffic.

5.1 Timeline (High Level)

  • Detection – Security log alerts for repeated 4625 failures from a single IP.
  • Analysis – Confirmed malicious RDP attempts and unusual outbound connections.
  • Containment – Disabled the targeted account and blocked the attacker IP at the firewall.
  • Eradication – Anti-malware scan and removal of any artifacts on the affected host.
  • Recovery – Restored any impacted files from backup, monitored for recurrence.
  • Lessons Learned – Hardened RDP exposure, lockout policy, and monitoring rules.

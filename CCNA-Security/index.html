CCNA Security / Network Defense – Capstone

Defense-in-depth design, traffic analysis, and incident response mapped to NIST and CIS controls.

CYBR-2102 CCNA Security Network Defense Fundamentals

Course Outcomes

Over this course I learned to:

  • Design segmented networks with external, DMZ, and internal zones.
  • Write and verify Cisco IOS ACLs that enforce least privilege between zones.
  • Capture and analyze traffic in Wireshark (ping sweeps, port scans, DoS-style floods).
  • Document findings in traffic analysis reports linked to CIS / NIST controls.
  • Write an incident response plan for brute-force RDP attacks with exfil attempts.

Milestones

Milestone 1

Defense-in-Depth Network Design

Packet Tracer design with external, DMZ, and internal zones. Firewall ACLs control web access, admin management, and DMZ ↔ internal traffic. Local firewalls and an access-control matrix tie network rules back to user roles.

Milestone 2

Traffic Analysis Report (Wireshark)

Captured live traffic and identified ping sweeps, SYN scans, and high-rate ICMP floods. Used Wireshark filters to confirm attack behavior and recommended defenses mapped to CIS Controls and NIST guidance.

Milestone 3

Incident Response Plan

Full NIST SP 800-61 incident lifecycle for a brute-force RDP and data exfiltration scenario: preparation, detection/analysis, containment, eradication, recovery, and lessons learned.

Labs & Supporting Work

Alongside the milestones I completed six Packet Tracer / IOS labs covering secure device setup, VLANs and inter-VLAN routing, ACLs, VPN/remote access, and device hardening. These show my ability to move from design → configuration → verification on real Cisco-style gear.

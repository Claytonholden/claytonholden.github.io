Topic 1: Recon, Scanning & Vulnerability Analysis

The practical workflow: map the attack surface, enumerate services accurately, then research vulnerabilities and communicate risk clearly.

OSINT Nmap CVE/CWE/NVD CVSS

What I can do

  • Gather passive intel and validate it with light active checks.
  • Perform targeted Nmap scans (ports, service versions, scripts) and document findings.
  • Use CVE/CWE/NVD/CVSS sources to explain impact, affected versions, and mitigations.
  • Write notes that are usable by non-technical stakeholders (clear risk + recommendations).

Portfolio Evidence

Workflow

Recon → Scan → Validate → Report

My repeatable checklist for new targets.

1) Scope & rules of engagement
2) Identify targets (domain/IP/app)
3) Passive OSINT (DNS/WHOIS/subdomains)
4) Active scan (nmap scan strategy)
5) Enumerate services & versions (-sV, scripts)
6) Map to CVE/CWE + CVSS severity
7) Recommend mitigations + verification steps
Tooling

Nmap “go-to” commands

Templates I use depending on goals.

# Fast top ports + versions
nmap -sV --top-ports 1000 TARGET

# Full TCP scan (slower)
nmap -p- -sV TARGET

# Common scripts
nmap -sC -sV TARGET

# Vuln research workflow (example)
nmap -sV --script vulners TARGET
Research

CVE / CWE / NVD / CVSS

How I move from “finding” → “what it means.”

CVE: specific vuln identifier
CWE: weakness category (root cause pattern)
NVD: enrichment database (impact + references + scoring)
CVSS: severity scoring (base metrics + vector)

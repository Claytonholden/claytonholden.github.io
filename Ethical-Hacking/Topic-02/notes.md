# Topic 2 — Social Engineering & Web/Client Attacks (Labs)

## What I practiced (controlled labs)
- SET: Credential Harvester concept (cloned login page captures form fields)
- BeEF: Hooked browser concept (actions inside browser context)

## Key ideas
- Users often don't notice URL changes or redirects.
- MFA + password managers reduce real-world impact.
- Defensive controls:
  - Security awareness training (URL/domain checking)
  - MFA everywhere
  - Browser hardening + patching
  - Secure headers/CSP, SRI
  - Logging + alerting on suspicious redirects

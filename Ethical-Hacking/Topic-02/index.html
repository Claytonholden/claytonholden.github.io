Topic 2: Social Engineering & Web/Client Attacks (Controlled Labs)

Social engineering is usually the easiest way into a network. These labs demonstrate how credential capture and browser-driven attacks work — and how defenders prevent them.

SET (Credential Harvester) BeEF (Browser Hook) User Awareness Hardening

What I can do

  • Explain phishing-style flows and what “credential harvesting” looks like in logs and traffic.
  • Demonstrate browser risk in a controlled lab environment and document the impact clearly.
  • Recommend mitigations: MFA, password managers, user training, browser hardening, logging, URL filtering.
  • Write professional, non-hype reports: what happened, why it matters, how to reduce risk.

Key Concepts (portfolio-safe)

SET Lab Concept

Credential Harvesting (Controlled)

Demonstrates how a cloned login page can capture submitted form fields and then redirect to the legitimate site—user thinks nothing happened. 

Defensive takeaways:
- Train users to check domains carefully
- MFA reduces impact of stolen passwords
- Monitor DNS/URL filtering and suspicious redirects
- Use password managers (domain matching)
BeEF Lab Concept

Hooked Browser ≠ “Hacked PC”

BeEF shows what an attacker can do inside the browser context after a user loads a malicious script—useful for understanding web risk and defense. 

Defensive takeaways:
- Patch browsers + extensions
- Reduce risky plugins/extensions
- CSP, SRI, and secure headers
- Network segmentation + egress controls
- Endpoint protections + logging
Professional Skill

How I write this up

Clear, employer-friendly format.

1) Scenario (controlled lab)
2) Observed behavior (what user sees vs what attacker gets)
3) Evidence (screenshots/log snippets)
4) Risk (impact + likelihood)
5) Mitigations (specific + prioritized)
6) Verification steps
